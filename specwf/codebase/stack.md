## Build Configuration

- **Bundler:** tsup — ESM output only, shebang banner `#!/usr/bin/env node`, declaration + declarationMap
- **Entry point:** `src/cli.ts` → `dist/cli.js`
- **Module resolution:** `bundler` (tsup-native, no path alias resolution at runtime)
- **Path aliases:** `@/*` → `src/*` — defined in tsconfig but currently unused in source

## Testing

- **Framework:** Vitest 4
- **Environment:** `node`
- **Test locations:** `tests/core/*.test.ts`, `tests/parser/*.test.ts`, `tests/integration/*.test.ts`
- **Co-location:** Source-adjacent tests not yet adopted despite convention stating "co-located with source"

## Platform

- **Primary platform:** OMP (Oh My Pi) — commands, agents, skills, hooks
- **Planned:** Claude Code (`.claude/commands/`, `.claude/agents/`)
- **Package manager:** npm (no lockfile alternative configured)

## CI/CD

- **GitHub Actions:** `publish.yml` — runs `npm run build && npm publish` on release tags

## Publishing

- **Entry:** `bin/specwf.js` (shebang script, gitignored, generated by tsup via `publicDir`)
- **Files shipped:** `dist/`, `bin/`, `README.md`
